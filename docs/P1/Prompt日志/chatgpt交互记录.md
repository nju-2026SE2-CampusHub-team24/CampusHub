# 人与 ChatGPT 交互日志整理（P1 阶段补充）

## 原始交互记录

### Prompt
> 怎么展开p1阶段的工作

### AI 回复摘要

P1 阶段是需求分析阶段，重点不是写代码，而是把“系统到底要做什么”弄清楚、写规范、留痕完整。本阶段主要需要完成以下工作：

1. 真实用户调研  
   - 设计问卷  
   - 设计访谈提纲  
   - 收集问卷结果与访谈笔记  

2. 完成 “AI 需求盲区挑战”  
   - 先让 AI 生成需求规格说明书  
   - 再根据人工调研独立整理需求  
   - 最后对比 AI 输出与人工调研结果，分析 AI 正确覆盖、遗漏、捏造和歧义的需求  

3. 编写正式需求规格说明书  
   - 按 IEEE 830 结构组织  
   - 包括引言、总体描述、功能需求、非功能需求、需求追踪矩阵  
   - 每条需求标注来源：[AI生成] / [人工补充] / [AI生成+人工修改]

4. 完成任务 3：用例建模与用户故事  
   - 用例图  
   - 3 个核心详细用例  
   - 至少 15 条用户故事  
   - 验收标准和优先级排序  

5. 完成需求评审、AI 协作反思日志 #1 和 Prompt 日志

### Prompt
> p1 的任务3具体是什么，能不能展开讲讲

### AI 回复摘要

P1 的任务 3 是“用例建模与用户故事”，分为两部分：

1. 任务 3.1：用例图  
   - 绘制系统级用例图  
   - 至少覆盖 3 个角色、15 个用例  
   - 编写 3 个核心用例的详细描述  
   - 详细用例需要包含：前置条件、基本流、替代流、后置条件  

2. 任务 3.2：用户故事  
   - 至少写 15 条  
   - 格式：作为 [角色]，我希望 [功能]，以便 [价值]  
   - 每条用户故事要有验收标准  
   - 用 MoSCoW 方法进行优先级排序（Must / Should / Could / Won't）

并建议 CampusHub 的核心角色可优先选为：
- 需求方
- 服务方
- 管理员

核心用例可优先围绕：
- 发布需求
- 接单 / 确认订单
- 完成订单并评价

### Prompt
> p1需求文档中要求的 Prompt 日志，是我们原始的和 ai 交互的记录吗

### AI 回复摘要

是的，Prompt 日志本质上就是本阶段与 AI 的原始交互记录。  
更准确地说，它应包含：

- 输入给 AI 的 prompt
- AI 的输出结果
- 最好标明对应的是哪项任务，例如：
  - 需求发散
  - 问卷设计启发
  - 需求文档生成
  - 用例生成
  - 用户故事生成

同时建议提交方式采用：

- 保留原始交互记录
- 再增加一份简短说明文档，说明每份记录分别对应 P1 的哪项任务

这样更方便老师和助教理解你们本阶段 AI 的实际使用情况。
